# 智慧城市：多路口路網與分散式強化學習控制 (Phase 2)

本計畫為 Smart Traffic Light Controller 的第二階段升級，將單一十字路口擴充為 2x2 城市路網，並實作具備「合作機制」的多代理人強化學習 (Cooperative MARL)。

## 決策與設定
1. **路網規模**：2x2 (四個相連的十字路口)。
2. **車輛轉向**：車輛在抵達路口時，有一定機率會隨機左右轉或直行。

## 系統架構變更

### 1. 核心邏輯 (TrafficSimulation)
- 將地圖改為圖論 (Graph) 結構。包含 4 個路口節點 (Nodes)。
- 道路 (Edges) 連接節點，具有不同的距離 (Length)。
- 車輛 (Cars) 生成於路網邊緣，行駛至路口時依據機率決定轉向，並排入對應的車道列隊。

### 2. 強化學習與綠波 (Green Wave) / 濾波連動機制
為了解決各路口「各自為政」導致的死結問題，我們將強化學習模型從獨立學習升級為**合作型多代理人強化學習 (Cooperative MARL)**，透過以下兩大機制達成「綠波/濾波連動」：

- **擴展視野 (State Augmentation)**：
  - 各路口除了觀察自己的 [目前相位, 南北雍塞度, 東西雍塞度] 之外，還會**額外觀察相鄰兩個路口的相位 (Phase)**。
  - 例如 `n00` 能夠看到東邊 `n10` 與南邊 `n01` 目前亮什麼燈，藉此預判車流，提前準備切換燈號。
  - State 維度將從 5 維擴充至 7 維。

- **共享利益 (Reward Blending)**：
  - 將各路口的 Reward 計算方式修改為「利益共同體」。
  - 核心公式：`最終得分 = 自己路口的 Local Reward + 0.5 * (相鄰路口的 Local Reward 總和)`。
  - 如果某個路口為了自己清空車流，卻把隔壁路口塞爆，該路口也會連帶受到懲罰，藉此迫使 AI 學習同步燈號。

### 3. 視覺化 (3D Intersection)
- 渲染 2x2 的網格狀街道。
- 攝影機預設拉遠，以城市俯視角展示整體車流。
- UI 將提供訓練圖表，可即時觀察「綠波/濾波」連動機制啟動後，整體路網壅塞度的下降趨勢。
